Research - Code smell detection aided by a decision tree classifier.

Code smell is an implementation choice that violates the well-known principles of cohesion, abstraction and separation of concerns on software development.

There is a commonplace about the threats on software quality caused by code smells, at the same time that there is a conflicted sense about what is a "smelliness code" though. In literature, the different types of code smell stands only in an informal definition of what it could be, thus every initiative of tracking smells on source code is under the influence of developer's experiences and backgrounds.

The subjectiveness around code smells was addressed by recent studies to empirically evaluate how each developer perceive the presence of code smell in code. These empirical studies were aimed to expose developers to software code snippets and gather their point of view about the occurrence of certain types of code smells.

This experiment follows the same research premise: we intend to expose developers to different source code snippets, written in java, all of them named as "smelly code" due to prediction provided by a machine learning predictor. But, unlike previous studies, we will test a different perspective - show a java code with a visual representation of a decision tree classifier. As an interpretable machine learning technique, this classifier, through its nodes and rules, will give insights to developers aimed to serve as a support for decision making, i. e., it explains how the predictor detected the presence of some type of code smell in code. Therefore, we expect that such visual representation of the rules may influence the subjectivity during code smell evaluation.

The results of the survey may be reported in academic publications. If you have any questions or concerns, please contact the authors.
